# BUG-027 — inferred_state taxonomy mismatch — no display map for computed states

**Status:** FIXED  
**Severity:** MEDIUM  
**Category:** ui / state display  
**Session:** S41  
**Commit:** 62f4439

---

## Symptom

After `compute_inferred_states.py` ran, topic rows in the IES dashboard showed no color dot or state label. The dot rendering logic in `dashboard.html` had no CSS class or label for the states the script was writing.

## Root Cause

`compute_inferred_states.py` was written with a separate taxonomy: `READY`, `CRUNCH`, `SHAKY`, `LEARNING`. These values do not appear in `gap_states.state CHECK(state IN ('UNVISITED','FLAGGED','IN_STUDY','VERIFIED','DECAYING'))` — the constraint is on the `state` column, but `inferred_state` is a separate column (m036/m037) with its own CHECK that matched the canonical gap_states values. The script was writing non-canonical values into `inferred_state`, violating its CHECK constraint and being silently rejected by SQLite (INSERT OR REPLACE treating CHECK failures as IGNORE). The dashboard template's color-dot map had no entry for the non-canonical values.

## Fix

Aligned `inferred_state` taxonomy in `compute_inferred_states.py` to the canonical set (`UNVISITED / FLAGGED / IN_STUDY / VERIFIED / DECAYING`) used by the rest of the system. Mapping:

| Script state (old) | Canonical state (new) |
|---|---|
| `READY` | `VERIFIED` |
| `SHAKY` | `FLAGGED` |
| `LEARNING` | `IN_STUDY` |
| `CRUNCH` | `IN_STUDY` (high priority flag) |

## Files Changed

- `scripts/compute_inferred_states.py` — `infer_state()` function return values
