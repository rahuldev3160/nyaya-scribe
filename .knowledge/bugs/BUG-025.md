# BUG-025 — Quiz model answer panel always empty

**Status:** FIXED  
**Severity:** HIGH  
**Category:** data / missing SELECT  
**Session:** S29  
**Commit:** 49d0d0e

---

## Symptom

After submitting a descriptive answer, the "Model Answer" panel appeared blank — no INTRODUCTION, BODY, or CONCLUSION text. The "Your Answer" panel rendered correctly.

## Root Cause

`get_questions()` in `db.py` joins `model_answers` but only selects `ma.answer_id` (used to filter questions that have answers). It does **not** select `ma.intro_text`, `ma.body_text`, `ma.conclusion_text`.

The template checks `{% if selected_q.intro_text %}` which is always falsy because these keys are not in the dict `get_questions()` returns.

Loading all three text columns for all 1219 questions on every quiz page view would be ~3MB per request — not acceptable.

## Fix

In `ies_quiz_bp.py`, after `selected_q` is resolved (single question), call `get_answer(conn, selected_q["question_id"])` and merge the three text fields into `selected_q`:

```python
if selected_q and selected_q.get("answer_id"):
    ma = get_answer(conn, selected_q["question_id"])
    if ma:
        selected_q = {**selected_q,
                      "intro_text": ma.get("intro_text") or "",
                      "body_text": ma.get("body_text") or "",
                      "conclusion_text": ma.get("conclusion_text") or ""}
```

One indexed lookup (`WHERE question_id=? AND exam_id=?`) only for the selected question.

- `web/blueprints/ies_quiz_bp.py` — import `get_answer`, merge text fields after question selection
