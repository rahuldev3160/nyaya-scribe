# BUG-026 — self_rating TEXT * 2.0 always returns 0

**Status:** FIXED  
**Severity:** HIGH  
**Category:** scoring / silent data corruption  
**Session:** S41  
**Commit:** 62f4439

---

## Symptom

Readiness scores on the IES and UPSC dashboards were always 0 even for users with multiple submitted and self-rated quiz attempts. `compute_inferred_states.py` reported UNVISITED for all topics regardless of actual attempt history.

## Root Cause

`self_rating` is stored in `descriptive_attempts` as TEXT: `'got_it'`, `'partial'`, or `'missed'`. All SQL that computed score metrics used arithmetic directly on the column — e.g. `AVG(self_rating * 2.0)`. SQLite coerces non-numeric TEXT to 0 in arithmetic expressions, silently returning 0 with no error. The bug existed in 5 separate SQL locations across 3 files.

## Fix

Replace all arithmetic on `self_rating` with a CASE expression that maps TEXT → numeric before computing:

```sql
CASE self_rating
    WHEN 'got_it'  THEN 8.0
    WHEN 'partial' THEN 5.0
    WHEN 'missed'  THEN 2.0
    ELSE NULL
END
```

The canonical mapping (also used in `compute_inferred_states.py`): `got_it=8`, `partial=5`, `missed=2`.

## Files Changed

- `web/db.py` — readiness score aggregate queries
- `web/blueprints/dashboard_bp.py` — topic avg_score computation
- `web/blueprints/upsc_dashboard_bp.py` — topic avg_score computation
- `scripts/compute_inferred_states.py` — infer_state() scoring logic
