## BUG-028 — UPSC Dashboard 500: Missing self_rating Column

**Status:** FIXED  
**Severity:** CRITICAL  
**Category:** schema-drift  
**Session:** S42  
**Commit:** 55f8a83  

### Symptom
`/upsc` returns 500 Internal Server Error on production Railway after S41 deploy.

### Root Cause
`m016_descriptive_self_rating.py` added `self_rating TEXT` to `descriptive_attempts` in `ies.db` only (`DB = "ies"`). No companion migration was created for `upsc.db`.

S41 added a hero strip to the UPSC dashboard with three queries that all reference `da.self_rating`:
- `_get_recommended_question` — `CASE da.self_rating WHEN 'got_it' THEN 8.0 ...`
- `_compute_readiness_score` — same CASE expression
- `attempt_stats_rows` — same CASE expression

SQLite throws `no such column: da.self_rating` on first query hit → Flask 500.

### Fix
`migrations/m038_add_self_rating_upsc.py` (`DB = "upsc_eco_opt"`): `ALTER TABLE descriptive_attempts ADD COLUMN self_rating TEXT`. Auto-applied on Railway deploy.

### Wider Pattern
**Schema drift risk:** Any migration adding a column to a shared-schema table (`descriptive_attempts`, `gap_states`, `user_mastery`, `return_quiz_questions`) must also add it to all exam DBs that have that table. Always check the full DB list when writing schema migrations.

**Related:** [[DECIDE-23]] (separate DB per optional means schema changes must be replicated manually)
